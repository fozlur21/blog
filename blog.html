Question-1: What is DOM and what is the purpose of DOM?

What is DOM:

The Document Object Model (DOM), a programming interface, represents the hierarchy of an HTML document's components as a tree-like hierarchical structure. It enables programmers to use scripting languages like JavaScript to dynamically change the information, structure, and style of a web page.

The purpose of DOM:

The DOM is designed to give programmes and scripts a platform- and language-neutral means of accessing and updating the content and structure of web pages. Being a tree of nodes and objects that can be accessed and modified using a standardised set of methods and properties, the DOM offers a standard way to describe HTML.

Name of the DOM methods:

getElementById(): Selects an element with a specific ID attribute.
getElementsByClassName(): Selects elements with a specific class name.
getElementsByTagName(): Selects elements with a specific tag name.
querySelector(): Selects the first element that matches a specific CSS selector.
querySelectorAll(): Selects all elements that match a specific CSS selector.
createElement(): Creates a new element with the specified tag name.
appendChild(): Adds a new child node to an element.
removeChild(): Removes a child node from an element.
replaceChild(): Replaces a child node with a new node.
innerHTML(): Gets or sets the HTML content of an element.
style(): Gets or sets the CSS style of an element.
addEventListener(): Attaches an event listener to an element.
removeEventListener(): Removes an event listener from an element.
getAttribute(): Gets the value of a specific attribute of an element.
setAttribute(): Sets the value of a specific attribute of an element.

Question-3:What is event bubble?

Event bubble:

When an event is triggered on a child element, it will propagate upward through its parent elements, activating their corresponding event handlers as well. This process is known as event bubbling. If not regulated, it can have unwanted repercussions. It can be handy for processing events on parent components that contain numerous child elements with the same event handler.

Question-4: What is a call back function and why will you use it?

call back function:

A callback function is one that receives another function as an input and is designed to be called after the first function has finished its job. A callback function's main function is to enable the calling function to carry out some logic after the called function has finished its job.

I will use it because:

Callback functions are frequently used in asynchronous programming, event handling, and generalisation to provide a means for a function to transfer the execution of certain logic to another function.
